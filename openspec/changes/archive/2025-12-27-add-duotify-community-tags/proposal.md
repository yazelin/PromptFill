# Proposal: add-duotify-community-tags

## Summary
在「我的範本」側邊欄新增「多奇」和「社群」兩個標籤分類，用於區分付費會員專屬範本和社群貢獻範本。

## Problem
目前的標籤系統只有基於範本內容分類（建築、人物、攝影等），無法區分：
1. 多奇教育訓練會員專屬的範本
2. 社群使用者貢獻的範本

## Solution
1. 新增「多奇」標籤：
   - 位置：排在「全部」之後、「社群」之前
   - 外觀：VIP 風格設計（漸層金色/紫色、光暈效果、特殊邊框），突顯這是付費功能
   - 行為：點擊後正常篩選顯示標記為「多奇」的範本
   - 範本內容：多奇專屬範本的內容會說明需要成為多奇學員才能使用，並引導至會員專屬頁面

2. 新增「社群」標籤：
   - 位置：排在「多奇」之後、原有標籤之前
   - 外觀：普通標籤樣式
   - 行為：點擊後篩選顯示標記為「社群」的範本

3. 自動標籤邏輯：
   - 從 GitHub Issue 同步的範本自動加入「社群」標籤
   - 使用者透過「提交給作者」送出範本時，若未包含「社群」標籤則自動加上

## Scope
- 修改 `TEMPLATE_TAGS` 新增「多奇」和「社群」
- 修改 `TemplatesSidebar.jsx` 調整標籤順序和「多奇」標籤的 VIP 外觀
- 修改 `github.js` 讓同步的範本自動帶「社群」標籤
- 修改 `App.jsx` 中提交範本的邏輯，自動加入「社群」標籤

## Out of Scope
- 多奇專屬範本的具體內容設計（由使用者另行建立）
- 會員驗證機制（範本內容直接說明並提供連結）
